POSIX / Linux Minimal Build Required Tools

The list of tools below is the minimal set required to build OpenSplice.

The following tools should be installed and available on the machine executable / library search path.

bash
shell
gcc
3.2.x or above (not needed if using Sun Studio)
Sun Studio
At this time only v12 of this compiler has been tested (not needed if using gcc)
gmake
3.80 or above.
flex
Any version is acceptable.
bison
Any version is acceptable.
perl
Version 5.8 or above is required.
Java SDK
This is optional. Oracle SDK v1.6 or above is acceptable. If omitted no Java APIs or tools will be generated.
Doxygen
This is optional. If omitted the documentation for the C# and I.S.O. C++ APIs will not be generated. Version 1.8 or greater is recommended for ideal results. (v6 of OpenSplice onwards only)
Qt4
Highly optional. Its only use is to build the iShapes demonstration application into the distribution, which can be used to illustrate DCPS use and verify interoperability. (v6 of DDS Community Edition onwards only). See demos/iShapes/README.md.

POSIX / Linux Steps to Build OpenSplice from Source

  1. Clone the OpenSplice project.
  2. Set environment variables so that the above tools can be located:
  3. Optionally set JAVA_HOME - set to the directory that a Oracle SDK (>1.6) is installed in.
  4. All other tools will be located from the machine $PATH
  5. Source the OpenSplice configure script to set-up your build environment. N.B. You must source this script each time you wish to configure a new shell as a build environment. The configuration choices are not persistent.
  6. Choose the target platform configuration.
  7. Invoke make to compile OpenSplice
  8. Invoke make install to build a OpenSplice Distribution

Windows Steps to build OpenSplice from Source

  1. Clone the OpenSplice repository.
  2. In a Cygwin shell set environment variables so that the required build tools can be located.
    1. VS_HOME - set to the installation directory of Visual Studio 2008 or 2010.
    2. WINDOWSSDKDIR - set to the location of the Windows SDK directory.
    3. Optionally set JAVA_HOME - set to the directory that an Oracle SDK (>1.6) is installed in.
    4. All other tools will be located from the cygwin shell $PATH
  3. Unset TMP and TEMP (lower and upper case) otherwise you may encouter error 256 while building the OpenSplice ISO C++ API. This is a known compatibility issue between Cygwin and Visual Studio. This can be done permanently by editing your /etc/profile and commenting out the setting of the variables.
  4. Source the OpenSplice configure script to set-up your build environment. N.B. You must source this script each time you wish to configure a new shell as a build environment. The configuration choices are not persistent.
  5. Choose the target platform configuration. N.B - 32/64 bit debug (dev) or release (release) builds are available.
  6. Invoke make to build OpenSplice
  7. Invoke make install to build an OpenSplice Distribution
