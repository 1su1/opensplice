Supported Platforms

PrismTech supports a wide variety of platforms. An up-to-date list of platforms is provided on PrismTech's web site at www.prismtech.com . Additional platforms may be supported according to demand.

OpenSplice DDS consists of an HDE (Host Development Environment) and an RTS (RunTime System). The HDE is used to develop DDS applications. It includes tooling for code generation as well as the environment for executing OpenSplice DDS application programs. The RTS is an environment that is only used for executing the OpenSplice DDS application programs on the target environment. It excludes the tooling for generation of the OpenSplice DDS application programs.

Prerequisites

OpenSplice DDS Tuner

The OpenSplice DDS Tuner requires the Java Runtime Environment: SUN JRE 1.5.

OpenSplice DDS Configurator

The OpenSplice DDS Configurator requires the Java Runtime Environment: SUN JRE 1.5.

OpenSplice DDS DCG (DLRL Code Generator)

The OpenSplice DDS DCG requires the Java Runtime Environment: SUN JRE 1.5

Building your own Corba C++ API library

  • Corba C++ (CCPP): The OpenSplice DDS DCPS API for the C++ language binding with Corba cohabitation is delivered using a specific compiler and ORB as shown in the table above.
    The ORB 'OpenFusion' can be obtained from PrismTech ( www.prismtech.com ).
    To be able to use a different compiler and a different ORB with the OpenSplice DDS Corba C++ API, we deliver the source code for this language with the OpenSplice DDS distribution.
  • Stand-alone C++: The OpenSplice DDS DCPS API for the C++ language binding without Corba cohabitation is delivered using a specific compiler as shown in the table above.
    To be able to use a different compiler with the OpenSplice DDS StandAlone C++ API, we deliver the source code for this language with the OpenSplice DDS distribution.

The OpenSplice DDS distribution contains a directory custom_lib/ccpp containing the source code for building your own Corba C++ API library for an ORB of your choice and compiler of your choice. Located in this directory is a README file, describing how to generate this custom library.
Identically a directory custom_lib/sacpp containing the source code for building your own StandAlone C++ API library for a compiler of your choice. Again a README file is delivered, describing how to generate the custom library.

Switching to the next OpenSplice DDS version

OpenSplice DDS uses a versioning policy, which reflects the severity of the changes applied. The OpenSplice DDS version consists of 3 digits: major, minor and patch (in that order). The changes that can be expected per digit change:

  • Major: typically contains changes and/or additions to the DDS API either syntactic and/or behavioral. These releases are always accompanied with a migration guide detailing the DDS API changes and in case of behavioral changes it contains recommendations (if possible) on how to realize the old behavior.
    Actions to take:
    • Application code has to be adapted in case the application uses methods that have been changed syntactically or when the behavior has changed.
    • Install the same OpenSplice DDS version on all your nodes. Do not mix versions as the wire protocol might not be compatible.
    • Re-generate the code generated by the OpenSplice DDS IDL preprocessor and DLRL Code Generator. Consequently you have to re-compile the generated code.
    • Delete the persistent store.
  • Minor: contains changes in generated code and/or the wireprotocol and requires the following actions:
    • Install the same OpenSplice DDS version on all your nodes. Do not mix versions as the wire protocol might not be compatible.
    • Re-generate the code generated by the OpenSplice DDS IDL preprocessor and DLRL Code Generator. Consequently you have to re-compile the generated code.
    • Delete the persistent store.
  • Patch: contains bugfixes only and is compatible with any previous version of OpenSplice DDS with the same major and minor version number.
