Report ID. Description
5627
dds888 		role logout exit handling deadlock
When we perform a role logout on our system, applications are terminated. Terminating some applications result in a crash deadlocking other terminating applications. OpenSplice DDS release V3.3 is used. Loggin in again is not possible anymore.
Analysis showed that this is caused by a reentrancy problem when deleting shared datareaders. Only the last application referencing the shared datareader is allowed to delete the datareader. However in this case more than 1 application tried to delete the shared datareader.
Solution: the reentrancy problem causing more than 1 application to delete the shared datareader has been solved. by using
