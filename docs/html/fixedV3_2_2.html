Fixed Bugs and Changes not affecting API

Report ID. Description
5363
dds730 		Unresolved API symbol: DDS_SubscriberListener__alloc
Building and linking the following piece of code on Windows fails with an unresolved external error _DDS_SubscriberListener__alloc during linkage. This failure occurs for all Entity Listeners EXCEPT the DataReaderListener. 

#include 
int main(int argc, char *argv[])
{
  DDS_SubscriberListener *list = DDS_SubscriberListener__alloc();
  DDS_free(list);
  
  return 0;
}
Solution: Solved by adding dll import/export construction for the listener allocation functions.
5370
dds733 		Problem restarting the daemon on windows
OpenSplice 3.2.1 is used on windows. The daemon is started using ospl start. In the task manager the spliced, durablity and networking processes can be seen. An ospl-info log is generated showing everything is started. The daemon is then stopped using ospl stop. The daemon is then restarted using ospl start. An ospl-error.log is then generated with: 

### Report Message ###
Version     : V3.2.1
Type        : ERROR
Context     : DurabilityService
File        : e:\r\components\services\durability2\code\d_durability.c
Line        : 789
Code        : 0
Description : Could not connect to URI 'file://C:/PrismTech/OpenSpliceV3/HDE/x86.win32/etc/config/ospl.xml'.
Node        : lat400-9
Process     : 2584
Thread      : 2840
Timestamp   : 1202220854.125662898 (Tue Feb 05 14:14:14 2008)
If the log files are deleted and the spliced process killed (the durability and networking are not there) then it is possible to start all services again.
Solution: An initialisation problem in the splice daemon has been found and fixed.
