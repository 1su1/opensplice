This page contains a list of all bugfixes and changes incorporated in OpenSplice V3.2.1

Fixed Bugs and Changes not affecting API

Report ID. Description
dds498 A number of issues need to be solved in the DCPS Reference Manuals
The following things need to be added to the reference manuals:
  • Mention in the find_topic section that a topic obtained in a DomainParticipant this way cannot be written/read when the corresponding TypeSupport has not yet been registered to that same DomainParticipant.
  • In the Appendix about the SQL syntax, it is not clear whether Unix wildcards can be used instead of normal SQL wildcards or must be used instead of normal wildcards. The latter is true: SQL wildcards are not supported in OpenSplice.
  • Mention the fact that the register and unregister calls can also be blocking calls that may return a RETCODE_TIMEOUT.
  • Appendix A does not mention a default value for the service_cleanup_delay in the Default Qos tables.
Solution: DCPS Reference manuals have been updated.
dds687 On windows the timestamps only have resolution of 1 millisecond
On the windows platform the timestamps (like the insertion time stored in the SampleInfo attribute) have a resolution of 1 millisecond
Solution: A new clock has been implemented, which has a resolution of 1 microsecond.
dds697 Version number of manuals incorrect
All manuals contain incorrect version number V3.1, while it should be V3.2. However no changes to the manuals have been made between V3.1 and V3.2
Solution Version number has been changed to V3.2
5313
dds702 		networking reaches defragmentation limit of 20000 and in that case uses 100% CPU
When the networking service reaches the limit of defragmentation buffers (configured to 20000) it uses 100% CPU load.
Solution An error in the cleanup algorithm of the defragmentation buffers caused the networking service to use all available CPU power. This error has been fixed.
5315
dds704 		durability crashes on a node during system startup
The durability service sometimes crashes during system startup
Solution A concurrency problem in the service has been fixed.

Fixed Bugs and Changes affecting API

No changes were made affecting the API.
